Ultima

Notifiche FCM via backend

Il server FastAPI esegue un price checker ogni 60 secondi usando MarketDataRegistry e invia notifiche push istantanee via Firebase Cloud Messaging. Niente più WorkManager Android: tutto è server-side, eliminando problemi con battery optimization e doppie notifiche.

Notifiche in foreground

Android non mostra automaticamente le notifiche FCM mentre l’app è aperta. Il listener pushNotificationReceived ora converte il messaggio push in notifica locale via CapacitorJS, così gli alert arrivano sempre, in qualsiasi stato dell’app.

Pannello diagnostica sviluppatore

Nuovo pannello in modalità sviluppatore con ping backend, stato FCM, conteggio dispositivi registrati e pulsante per inviare una notifica FCM di test in tempo reale senza aspettare un trigger di prezzo.

Allarmi di range con cooldown

Nuovo tipo di allarme: corridoio di prezzo min-max con notifica al cambio di stato (entrata/uscita). Il cooldown di 5 minuti evita spam su prezzi che oscillano sul confine del range.

Fix download APK in-app (Android 13+)

Lo spinner di download rimaneva bloccato su Android 13+. Il broadcast del sistema DownloadManager richiede RECEIVER_EXPORTED: il flag corretto permette ora di ricevere la notifica di completamento dal processo di sistema.

Diagnostica backend con CapacitorHttp

Le chiamate HTTP dalla WebView venivano bloccate da Android (cleartext + CORS). Sostituendo fetch() con CapacitorHttp (OkHttp nativo) le richieste bypassano le restrizioni WebView e i messaggi di errore sono ora specifici: timeout, connessione rifiutata, rete non raggiungibile.

Roadmap

CryptoSentinel AI evolve in un agente di trading autonomo su BNB Smart Chain. Gli alert di prezzo già esistenti diventano il trigger dell’agente.

Step 3 — Prossimo

Dati Multi-Provider

  • CoinMarketCap (piano Startup) come provider primario
  • CoinGecko come provider secondario selezionabile
  • Astrazione MarketDataProvider — formato interno normalizzato
  • Rate limiting + cache crediti CMC
Step 4 — Esecuzione

Layer di Trading

  • Spot → Trust Wallet Agent Kit (TWAK), signing autonomo
  • Perpetual → BNB AI Agent SDK / EIP-712
  • Pagamenti dati → x402 su BSC
  • Gas management + multi-RPC BSC fallback
Step 6 — Il cuore

Agente AI Autonomo

  • Claude come meta-controller (poteri limitati, decisionale)
  • Spot: momentum + VWAP + volume relativo + ATR
  • Perp: Volume Profile mean reversion (POC, VAH, VAL)
  • Risk management: stop ATR, daily loss limit, drawdown cap
Step 8 — Osservabilità

Dashboard Web Unificata

  • Monitoring completo + 3 viste agente (Spot / Perp / Globale)
  • System Health (servizi, carburante, drawdown)
  • Demo/Replay trade-by-trade con prove on-chain
  • Export spiegazione strategia per i giudici
